# MTProto Proxy Manager

Установщик и менеджер личного MTProto-прокси для Telegram на базе [9seconds/mtg](https://github.com/9seconds/mtg) с поддержкой нескольких клиентов (свой секрет на каждое устройство).

## Что делает

Поднимает на VPS отдельный экземпляр mtg для каждого устройства (`mtg@<name>.service` через systemd template) с собственным портом, секретом и Fake-TLS-доменом. Управление — через единое CLI-меню `tgproxy`.

## Возможности

- Установка mtg последней версии с проверкой SHA256
- Несколько клиентов одновременно — у каждого свой секрет, порт, Fake-TLS-домен
- Генерация `tg://proxy` и `https://t.me/proxy` ссылок, плюс QR-код в терминале
- Импорт уже существующего секрета (формат `ee...hex`) — старые устройства продолжают работать без перенастройки
- Перезапуск, обновление, удаление через меню
- BBR + ip_forward для скорости
- UFW автоматически открывает порты клиентов (если активен)
- Systemd-сервисы с hardening: `User=mtg`, `NoNewPrivileges`, `ProtectSystem=strict`, `ProtectHome`, `PrivateTmp`
- Конфиги клиентов с правами `600`, owner `mtg:mtg`

## Установка

На Ubuntu/Debian VPS под root:

```bash
curl -fsSL https://raw.githubusercontent.com/78ers/mtproto/main/install.sh -o install.sh
chmod +x install.sh
sudo ./install.sh
```

После установки скрипт доступен как команда `tgproxy`.

## Использование

```
  ╔══════════════════════════════════════════╗
  ║      MTProto Proxy Manager (mtg)         ║
  ╚══════════════════════════════════════════╝

  1) Добавить клиента
  2) Управление клиентами
  3) Служба
  0) Выход
```

### 1) Добавить клиента

Запрашивает имя, автоматически выбирает свободный порт из диапазона `8443-8500`, спрашивает источник секрета:

1. Сгенерировать (Fake-TLS под `www.google.com`)
2. Сгенерировать с другим Fake-TLS доменом
3. Ввести готовый секрет в формате `ee...hex` — удобно для переноса существующих настроек

После создания сразу показывает `tg://`-ссылку и QR.

### 2) Управление клиентами

Показывает таблицу клиентов с портами, статусом и Fake-TLS-доменом, после выбора:

- Показать ссылку + QR
- Переименовать
- Сменить секрет (с предупреждением, что старые ссылки сломаются)
- Удалить
- Логи (последние 50 строк из journalctl)

### 3) Служба

- Перезапустить mtg (все клиенты)
- Перезагрузить VPS (с подтверждением `YES`)
- Обновить mtg (проверяет последний релиз, обновляет, перезапускает клиентов)
- Полное удаление (требует ввода `YES`)

## Где что лежит

| Путь | Назначение |
|---|---|
| `/usr/local/bin/mtg` | бинарь mtg |
| `/usr/local/bin/tgproxy` | сам скрипт-менеджер |
| `/etc/mtg/clients/<name>.toml` | TOML-конфиг клиента (`secret` + `bind-to`) |
| `/etc/systemd/system/mtg@.service` | systemd template, запускает `mtg run /etc/mtg/clients/%i.toml` |

Пример минимального TOML-конфига:

```toml
secret = "ee473ce5d4958eb5f968c87680a23854a0676f6f676c652e636f6d"
bind-to = "0.0.0.0:8443"
```

## Подключение из Telegram

Откройте в Telegram → Настройки → Данные и память → Прокси → Добавить прокси → MTProto, либо просто нажмите `tg://proxy?...` ссылку, либо отсканируйте QR.

## Совместимость со старыми секретами

Скрипт принимает готовые секреты вида `ee<16 байт случайных>ff<hex-домен>`. При импорте автоматически извлекает Fake-TLS-домен и сохраняет всё как есть — `tg://proxy` ссылки на существующих устройствах продолжают работать без изменений.

## Безопасность

- SHA256-проверка бинаря mtg при скачивании из GitHub-релиза
- Конфиги клиентов с правами `600`, владелец `mtg:mtg`, директория `700`
- Systemd-юниты с hardening (см. выше)
- UFW: скрипт **не активирует** UFW сам и **не меняет** дефолтные политики — добавляет только `allow <port>/tcp` для портов клиентов, если UFW уже активен
- IP_forward и BBR включаются через `/etc/sysctl.conf` (полезно, безопасно)

## Зависимости

Скрипт сам ставит через `apt`: `curl`, `wget`, `jq`, `qrencode`, `xxd`, `ca-certificates`.
